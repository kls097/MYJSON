## 二、功能需求

### 2.1 智能合并 (Smart Merge)

#### 2.1.1 功能描述

合并两个或多个 JSON 对象，支持多种合并策略，智能检测冲突并提供可视化解决方案。

#### 2.1.2 合并策略

| 策略名称 | 描述 | 适用场景 |
|---------|------|---------|
| **深度合并 (Deep Merge)** | 递归合并嵌套对象，合并数组元素 | 配置合并、数据聚合 |
| **覆盖合并 (Override)** | 右侧对象值覆盖左侧 | 配置覆盖、版本更新 |
| **数组合并 - 追加 (Append)** | 数组合并，保留所有元素 | 日志收集、审计记录 |
| **数组合并 - 去重 (Dedupe)** | 数组合并，去除重复元素 | 标签合并、分类合并 |
| **数组合并 - 替换 (Replace)** | 数组完全替换 | 列表更新 |

#### 2.1.3 输入输出

- **输入**: 左侧 JSON、右侧 JSON、合并策略选择
- **输出**: 合并后的 JSON、冲突列表（若有）

#### 2.1.4 冲突检测规则

当以下情况发生时，标记为冲突：
1. 同一路径下，左右两侧类型不同
2. 深度合并时，非对象类型被覆盖
3. 使用覆盖策略时，用户未确认覆盖

---

### 2.2 冲突解决 (Conflict Resolution)

#### 2.2.1 功能描述

在合并过程中，当存在无法自动解决的差异时，提供可视化界面让用户选择保留哪一侧的值。

#### 2.2.2 冲突类型

| 冲突类型 | 描述 | 默认处理 |
|---------|------|---------|
| **类型冲突** | 左侧和右侧类型不同 | 需用户选择 |
| **覆盖冲突** | 覆盖策略下的值差异 | 需用户确认 |
| **数组冲突** | 数组合并不能自动合并 | 需用户选择策略 |

#### 2.2.3 交互设计

- 冲突项高亮显示
- 提供"保留左侧"/"保留右侧"/"使用合并值"三个选项
- 支持"全部应用此规则"批量操作

---

### 2.3 三方合并 (Three-Way Merge)

#### 2.3.1 功能描述

比较三个 JSON：基础版本、左侧版本、右侧版本，生成合并结果。

#### 2.3.2 场景

- 代码合并（基础版本 → 我的修改 / 你的修改）
- 配置管理（默认配置 → 开发配置 / 生产配置）

#### 2.3.3 状态标记

| 状态 | 描述 |
|------|------|
| **未修改** | 三个版本相同 |
| **仅左侧修改** | 基础版本 vs 左侧有变化 |
| **仅右侧修改** | 基础版本 vs 右侧有变化 |
| **双方修改** | 左右两侧都有变化（可能冲突） |
| **冲突** | 两侧修改不同，需人工解决 |

## 四、技术设计

### 4.1 模块结构

```
src/
├── utils/
│   └── jsonMerger.js          # 新增：合并引擎
├── composables/
│   └── useJsonMerge.js        # 新增：合并逻辑
└── components/
    ├── JsonMergePanel.vue     # 新增：合并面板
    ├── ConflictResolver.vue  # 新增：冲突解决组件
    └── ThreeWayMergeView.vue # 新增：三方合并视图
```

### 4.2 核心 API

```javascript
// jsonMerger.js

/**
 * 深度合并两个 JSON 对象
 * @param {*} target - 目标对象
 * @param {*} source - 源对象
 * @param {Object} options - 合并选项
 * @returns {Object} { result, conflicts }
 */
export function deepMerge(target, source, options = {})

/**
 * 覆盖合并
 * @param {*} target - 目标对象
 * @param {*} source - 源对象
 * @returns {Object} { result, conflicts }
 */
export function overrideMerge(target, source)

/**
 * 数组合并
 * @param {Array} target - 目标数组
 * @param {Array} source - 源数组
 * @param {string} strategy - 策略: append/dedupe/replace
 * @returns {Array} 合并后的数组
 */
export function mergeArrays(target, source, strategy = 'append')

/**
 * 三方合并
 * @param {*} base - 基础版本
 * @param {*} left - 左侧版本
 * @param {*} right - 右侧版本
 * @returns {Object} { result, conflicts, status }
 */
export function threeWayMerge(base, left, right)
```

### 4.3 合并选项

```javascript
const defaultOptions = {
  arrayStrategy: 'append',     // append/dedupe/replace
  ignoreNull: false,          // 是否忽略 null 值
  ignoreEmpty: false,         // 是否忽略空对象/数组
  customMerge: null,          // 自定义合并函数
  detectConflicts: true       // 是否检测冲突
}
```
